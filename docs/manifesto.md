# Manifiesto de PostuLatte

> Este documento define la identidad permanente de PostuLatte.

Su propósito es establecer los principios fundamentales del proyecto, independientemente de los cambios tecnológicos, arquitectónicos o de implementación que puedan ocurrir con el paso del tiempo.

Toda decisión de arquitectura, diseño, implementación o experiencia de usuario deberá respetar los principios aquí definidos.

Si una nueva funcionalidad entra en conflicto con este manifiesto, deberá replantearse antes de ser implementada.

Este documento tiene prioridad sobre cualquier decisión técnica.

---

# 1. Nuestra razón de existir

PostuLatte existe para devolver tiempo a las personas.

Buscar empleo implica realizar una enorme cantidad de tareas repetitivas:

- leer ofertas;
- interpretar requisitos;
- adaptar currículums;
- redactar cartas de presentación;
- comparar oportunidades;
- organizar información.

Todas estas tareas consumen tiempo que podría dedicarse a prepararse mejor profesionalmente, aprender nuevas habilidades o simplemente disfrutar del tiempo libre.

PostuLatte automatiza el trabajo repetitivo.

Nunca automatiza el criterio humano.

---

# 2. La Inteligencia Artificial es una herramienta

La Inteligencia Artificial nunca reemplaza al usuario.

Su única función es eliminar trabajo repetitivo.

Toda decisión importante pertenece exclusivamente a la persona.

El usuario siempre conserva el control absoluto sobre:

- qué analizar;
- qué generar;
- qué guardar;
- qué modificar;
- qué eliminar;
- qué compartir.

La IA propone.

El usuario decide.

Siempre.

---

# 3. El usuario mantiene el control

PostuLatte nunca actuará por iniciativa propia cuando una acción pueda afectar:

- la privacidad;
- los datos personales;
- el almacenamiento;
- el hardware;
- la conexión a Internet.

Toda acción potencialmente relevante requerirá una acción explícita del usuario.

Esto incluye, entre otras:

- descargar modelos;
- utilizar proveedores externos;
- comprobar actualizaciones;
- eliminar información;
- modificar configuraciones importantes;
- realizar instalaciones.

El consentimiento explícito forma parte de la identidad del proyecto.

---

# 4. La privacidad es el comportamiento por defecto

Toda la aplicación está diseñada bajo el principio de privacidad por defecto.

Siempre que sea técnicamente posible:

- los documentos permanecerán en el equipo del usuario;
- los análisis se ejecutarán localmente;
- los modelos se ejecutarán localmente;
- la información no abandonará el equipo.

Ningún documento será enviado a servicios externos sin una decisión explícita del usuario.

La utilización de proveedores externos siempre será una elección, nunca una obligación.

---

# 5. Offline First

Todo aquello que pueda funcionar sin conexión a Internet debe funcionar sin conexión.

La conexión únicamente será necesaria cuando el usuario decida utilizar funciones que realmente dependan de Internet, como por ejemplo:

- descargar modelos;
- buscar actualizaciones;
- utilizar proveedores externos de IA;
- acceder a servicios remotos.

La ausencia de conexión nunca debe impedir el funcionamiento de las capacidades locales.

---

# 6. Transparencia absoluta

El usuario tiene derecho a saber qué está haciendo el programa.

PostuLatte nunca ocultará procesos importantes.

Siempre que una operación requiera tiempo, recursos o interacción relevante, el sistema deberá informar:

- qué está haciendo;
- por qué lo está haciendo;
- qué consecuencias tendrá.

La transparencia genera confianza.

La confianza forma parte del producto.

---

# 7. El software se adapta al usuario

El objetivo de PostuLatte es llegar a la mayor cantidad posible de personas.

No se diseñará pensando únicamente en hardware de gama alta.

Siempre que sea posible:

- el software elegirá configuraciones apropiadas;
- recomendará modelos adecuados;
- reducirá requisitos innecesarios;
- minimizará el consumo de recursos.

La experiencia debe adaptarse al equipo disponible.

Nunca al revés.

---

# 8. Simplicidad por encima de complejidad

Toda nueva dependencia deberá justificar su existencia.

Toda nueva funcionalidad deberá justificar su utilidad.

Toda nueva configuración deberá justificar su necesidad.

Si una solución sencilla resuelve correctamente un problema, deberá preferirse frente a una solución más compleja.

La complejidad únicamente es aceptable cuando aporta un beneficio real.

---

# 9. Calidad antes que velocidad

Una funcionalidad no se considera terminada únicamente porque funciona.

Para formar parte del proyecto deberá cumplir, cuando corresponda:

- documentación actualizada;
- arquitectura coherente;
- decisiones registradas mediante ADR;
- pruebas automatizadas;
- manejo adecuado de errores;
- tipado consistente;
- integración con el resto del sistema.

La calidad nunca será sacrificada por velocidad de desarrollo.

---

# 10. Documentación antes que implementación

Las decisiones importantes no comienzan escribiendo código.

El orden natural del proyecto es:

Diseñar.

↓

Documentar.

↓

Implementar.

↓

Probar.

↓

Versionar.

La documentación constituye la fuente oficial de conocimiento del proyecto.

El código implementa la documentación.

Nunca al revés.

---

# 11. El dominio representa la realidad

Los documentos originales son únicamente una fuente de información.

Una vez procesados, dejan de formar parte del flujo principal.

Todo el sistema trabaja sobre modelos del dominio.

El dominio representa la realidad del usuario.

Los documentos únicamente describen esa realidad.

---

# 12. Los datos pertenecen al usuario

Toda información aportada o generada para el usuario pertenece exclusivamente al usuario.

Esto incluye, entre otros:

- currículums;
- perfiles profesionales;
- ofertas almacenadas;
- resultados de análisis;
- cartas de presentación;
- documentos generados;
- futuras bases de datos que almacenen dicha información.

PostuLatte únicamente custodia estos datos para prestar su servicio.

Nunca adquiere propiedad sobre ellos.

---

# 13. La tecnología es reemplazable

Ninguna tecnología concreta define la identidad del proyecto.

Lenguajes.

Frameworks.

Modelos de IA.

Motores de base de datos.

Bibliotecas.

Todos ellos podrán cambiar con el tiempo.

La identidad de PostuLatte reside en sus principios.

Nunca en sus herramientas.

---

# 14. La evolución debe respetar la identidad

El proyecto evolucionará continuamente.

Se incorporarán nuevas capacidades.

Se eliminarán tecnologías obsoletas.

Se actualizarán modelos.

Se adaptará a nuevo hardware.

Pero toda evolución deberá respetar este manifiesto.

Una mejora técnica nunca justificará abandonar los principios fundamentales del proyecto.

---

# Conclusión

PostuLatte no pretende ser simplemente una herramienta basada en Inteligencia Artificial.

Pretende ser un asistente confiable, transparente, respetuoso con la privacidad y diseñado para devolver tiempo a las personas.

Su objetivo no es impresionar con tecnología.

Su objetivo es ayudar.

Siempre bajo una única condición:

La decisión final pertenece al usuario.
